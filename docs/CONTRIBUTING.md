# CONTRIBUTING.md

## Before Making Changes

Read:
1. `PROJECT_CONTEXT.md`
2. `AGENTS.md`
3. `REPO_MAP.md`
4. the relevant files in `architecture/`, `development/`, `design/`, and `product/`

## Working Style

- keep changes small
- avoid unrelated refactors
- respect platform vs game boundaries
- document major decisions when architecture changes
- update `TODO.md` and `CHANGELOG.md` when appropriate

## Review Philosophy

Changes should be:
- scoped,
- explainable,
- easy to validate,
- and aligned with the modular game philosophy.
