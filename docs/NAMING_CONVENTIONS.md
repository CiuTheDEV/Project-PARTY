# NAMING_CONVENTIONS.md

## General

- prefer explicit names
- avoid vague folders like `misc`, `stuff`, or `temp`
- use stable game slugs
- keep platform/shared/game naming obvious

## Recommended Terms

- hub
- game-module
- game-registry
- session
- mode
- screen
- shared-ui
- worker
- route

## Naming Rule

Only use terms like `host`, `controller`, `presenter`, or `viewer` when a specific game genuinely needs them.
Do not make them global repository vocabulary unless they are truly shared.
